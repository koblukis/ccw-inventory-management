Factory Inventory Management — Architecture

Vue 3 + FastAPI + In-memory JSON data  |  Full-stack demo application

Tech Stack

Frontend — port 3000
Vue 3 + Vite
  • Composition API
  • Vue Router 4.3
  • Axios (HTTP client)
  • Tailwind CSS
  • Custom SVG charts
Backend — port 8001
Python FastAPI
  • Uvicorn (ASGI)
  • Pydantic v2 validation
  • CORS enabled (*)
  • 20+ REST endpoints
  • In-memory filtering
Data layer
JSON files
  • 7 JSON files in server/data/
  • Loaded at startup
  • No database / no persistence
  • 200+ orders, 20+ inventory items
  • Tasks & POs in-memory only

Architecture Notes

No persistence

Tasks and purchase orders live only in server memory. Restarting the backend resets them. All other data comes from JSON files loaded at startup.

Filter state is global

useFilters() is a singleton composable — all 7 views share the same filter refs. Changing a filter in the navbar immediately re-queries every mounted view.

No authentication

CORS is open (*). useAuth.js tracks a mock user state client-side only. There is no token validation or session handling on the backend.

Revenue goal constants

Dashboard KPIs compare against hardcoded goals: $800K/month for a single month filter, $9.6M YTD when all months are selected.
